## Implemented

- Task model
- One-to-many relation: User hasMany Tasks
- Database migration
- TaskStatus enum
- Full CRUD for tasks
- Form Request validation
- TaskService for task CRUD operations
- Auth routes with guest/auth middleware
- Auth routes for tasks management
- Login view and controller
- Guest and main layouts
- Blade views with Tailwind CSS (responsive layout with sidebar)
- Tasks displayed in table format
- Modal confirmation for delete action
- Search by title (LIKE query)
- Filter by status
- Query filter classes (LikeSearchFilter, WhereFilter)
- FilterSearchService for applying filters
- TaskFactory for testing
- Feature tests:
    - Authenticated user can create task
    - User cannot update another user's task
- Integration test for search filtering
